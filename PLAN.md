# 개인 가계부 웹사이트 기획안

## 1. 프로젝트 목표

개인이 일상의 수입/지출을 쉽게 기록하고, 소비 패턴을 시각적으로 파악하여
더 나은 재정 관리를 할 수 있도록 돕는 웹 기반 가계부 서비스.

- 입력이 빠르고 귀찮지 않을 것 (3초 안에 한 건 입력)
- 한눈에 이번 달 재정 상태를 파악할 수 있을 것
- 모바일에서도 편하게 사용 가능할 것

---

## 2. 핵심 기능 (MVP)

가계부 앱의 본질. 이것만 있어도 서비스가 성립하는 기능들.

### 2.1 사용자 인증
- 이메일/비밀번호 회원가입 및 로그인
- 소셜 로그인 (Google, Kakao)
- 비밀번호 재설정

### 2.2 거래 내역 관리
- 수입/지출 기록 (금액, 날짜, 카테고리, 메모, 결제수단)
- 거래 내역 목록 조회 (날짜별/카테고리별 필터)
- 수정 및 삭제
- 빠른 입력 UI (최근 사용 카테고리/금액 제안)

### 2.3 카테고리 관리
- 기본 카테고리 제공 (식비, 교통, 쇼핑, 주거, 여가 등)
- 사용자 정의 카테고리 추가/수정/삭제
- 카테고리별 아이콘 및 색상

### 2.4 대시보드
- 이번 달 총수입/총지출/잔액
- 카테고리별 지출 비율 (원형 차트)
- 최근 7일 일별 지출 추이 (라인 차트)
- 이번 달 가장 많이 쓴 카테고리 TOP 3

### 2.5 월별 리포트
- 월별 수입/지출 요약
- 전월 대비 증감
- 카테고리별 월간 지출 총액

---

## 3. 확장 기능 (Phase 2)

MVP 이후 사용자 재방문율과 만족도를 높이는 기능들.

### 3.1 예산 설정
- 월별 전체 예산 설정
- 카테고리별 예산 설정
- 예산 초과/임박 시 시각적 경고 (색상, 프로그레스 바)

### 3.2 반복 거래
- 월세, 구독료 등 정기 지출 자동 등록
- 월급 등 정기 수입 자동 등록
- 반복 규칙 설정 (매월 n일, 매주 등)

### 3.3 자산 및 계좌 관리
- 현금, 은행 계좌, 신용카드 등 자산 등록
- 계좌별 잔액 관리
- 계좌 간 이체 기록

### 3.4 검색 및 고급 필터
- 키워드 검색 (메모 내용)
- 기간/금액 범위/카테고리 복합 필터
- 거래 내역 CSV/Excel 내보내기

### 3.5 태그
- 카테고리와 별개로 자유 태그 (#휴가, #친구모임)
- 태그별 지출 집계

---

## 4. 고급 기능 (Phase 3)

차별화 요소. 경쟁 서비스보다 한 발 앞서기 위한 기능.

### 4.1 데이터 시각화 강화
- 연간 트렌드 (월별 비교 막대 차트)
- 히트맵 (달력 형태의 일별 지출 강도)
- 카테고리별 지난 6개월 추이

### 4.2 목표 및 저축
- 저축 목표 설정 (여행, 비상금 등)
- 목표 달성률 추적
- 저축 챌린지 (n일간 커피 안 마시기 등)

### 4.3 인사이트 및 알림
- 이상 지출 감지 ("이번 주 외식비가 평균보다 많아요")
- 주간/월간 리포트 이메일 발송
- 예산 초과 알림

### 4.4 영수증 및 첨부
- 영수증 사진 첨부
- (선택) OCR을 통한 금액/날짜 자동 추출

### 4.5 데이터 백업 및 가져오기
- 데이터 전체 내보내기 (JSON, CSV)
- 은행/카드 거래 내역 CSV 가져오기
- (장기) 오픈뱅킹 API 연동

---

## 5. 비기능 요구사항

### 5.1 UX/UI
- 모바일 우선 반응형 디자인
- 다크 모드 지원
- 접근성 (WCAG 2.1 AA 기준)
- 키보드 단축키 (빠른 입력용)

### 5.2 성능
- 초기 로딩 3초 이내
- 거래 입력 응답 500ms 이내
- 월별 데이터 조회 1초 이내

### 5.3 보안
- HTTPS 전 구간 적용
- 비밀번호 해싱 (bcrypt 등)
- 세션/토큰 기반 인증
- 민감 데이터 암호화 저장 고려

### 5.4 국제화
- 한국어 우선, 영어 지원 고려
- 다중 통화 대응 (원화 기본)

## 6. 기술 스택 후보

선택 전 비교 검토가 필요한 영역. 지금 확정하지 않음.

- **프론트엔드**: React + TypeScript 또는 Next.js / Vue + Nuxt
- **백엔드**: Node.js (Express/Fastify/NestJS) 또는 Python (FastAPI)
- **데이터베이스**: PostgreSQL (관계형 데이터에 적합)
- **인증**: Auth.js(NextAuth) 또는 자체 JWT
- **차트**: Recharts, Chart.js, Apache ECharts
- **배포**: Vercel / Netlify (프론트), Railway / Fly.io (백엔드)

## 7. 개발 로드맵 (제안)

| 단계 | 기간 | 범위 |
|------|------|------|
| Phase 1 (MVP) | 4-6주 | 인증, 거래 CRUD, 카테고리, 대시보드, 월별 리포트 |
| Phase 2 | 3-4주 | 예산, 반복 거래, 계좌 관리, 검색/내보내기 |
| Phase 3 | 4-6주 | 고급 시각화, 저축 목표, 인사이트, 영수증 |

## 8. 고민이 필요한 지점

의사결정 필요.

- **범위**: 혼자 쓰는 개인용인가, 공개 서비스인가? (인증/인프라 규모가 달라짐)
- **데이터 민감도**: 금융 데이터 저장 방식과 암호화 수준
- **오프라인 지원**: PWA로 만들어 오프라인에서도 입력 가능하게 할지
- **가족 공유**: 한 계정에 여러 사용자 또는 가계부 공유 기능이 필요한지
- **수익 모델** (공개 서비스 시): 무료 / 프리미엄 / 구독
